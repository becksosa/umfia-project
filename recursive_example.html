How optimal_p_win works

A recursive lookup fills the entire probability table from just a handful of known outcomes. Disclaimer - this diagram is AI generated, but it does explain the process quite well.

1

Seed known outcomes

States where the winner is already decided get their probabilities filled in first. Everything else starts as None.

2

Recurse into the unknown

Call the function on a state with no probabilities. It rolls all 36 dice combos and calls itself on each result — drilling deeper until it hits a known state.

3

Bubble up and cache

Probabilities flow back up through the call chain. Each state computes p_roll and p_bank, writes them to the table, and returns max(roll, bank).

Every computed state is now cached — future lookups return instantly
